# LogBERT: Log File Anomaly Detection using BERT - An Explainer

## 1. Introduction
LogBERT is a self-supervised approach for log anomaly detection based on the Bidirectional Encoder Representations from Transformers (BERT). It aims to detect anomalies in logs generated by online systems by learning the baseline of normal log sequences and identifying deviations.

### Key Advantages:
- **Contextual Understanding:** Captures both left and right context of a log entry, unlike RNNs.
- **Sequential Insight:** Detects anomalies that may seem normal in isolation but are suspicious in a specific sequence.
- **No Manual Rules:** Learns patterns automatically from raw log data.

---

## 2. Training Framework

### Step 1: Log Parsing
Raw logs are unstructured. A parser (like Drain or Spell) extracts **Log Templates** (constant strings) from the variable noise (IPs, timestamps).
*   **Example Log Line:** `Invalid user admin from 192.168.1.1`
*   **Log Template:** `Invalid user <user> from <IP>`
*   **Event ID:** Each unique template is assigned a unique integer ID (e.g., `E12`).

### Step 2: Vocabulary Generation
Templates occurring at least twice (threshold defined as 1) are added to the vocabulary.

### Step 3: Sequence Creation
Logs are grouped into sequences using:
- **Sliding Window:** e.g., fixed number of logs or fixed time interval (5 mins).
- **Session IDs:** Logs belonging to the same user or process.

### Step 4: Special Tokens
A special `[DIST]` (Distance) token is added to the beginning of each sequence to help in clustering normal behaviors.

---

## 3. The LogBERT Model
LogBERT uses a Transformer Encoder with two main tasks during training:

### Task A: Masked Log Key Prediction (MLKP)
- A ratio of log keys in a sequence (e.g., 15%) are randomly replaced with a `[MASK]` token.
- The model learns to predict the original log key using the surrounding context.
- **Loss:** Cross-Entropy loss between predicted and actual keys.

### Task B: Volume of Hypersphere Minimization (VHM)
- The model maps normal log sequences to points in a high-dimensional space.
- It tries to minimize the distance between the `[DIST]` token representation and the center `c` of all normal sequences.
- This creates a tight "cluster" of normal system behavior.

---

## 5. Inference (Detection) Pipeline
Once trained on normal data, LogBERT detects anomalies as follows:

1.  **Masking:** Mask tokens in the test sequence.
2.  **Top-g Candidates:** Predict the masked tokens and take the top *g* most likely templates.
3.  **Validation:** If the *actual* log key is NOT in the top *g* candidates, that line is marked as an anomaly.
4.  **Sequence Score:** if there are more than *r* anomalous logs in a sequence, the entire sequence is flagged as an **Intrusion/Anomaly**.

---

## 6. Summary
LogBERT transforms log anomaly detection into a language modeling problem. By mastering the "language" of normal system logs, it becomes highly sensitive to the structural and temporal shifts that characterize cyberattacks and system failures.
